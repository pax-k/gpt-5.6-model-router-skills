# Model and effort calibration

Version 0.3 treats routing as cost control: start with the least expensive role adequate for ambiguity, breadth, verification, and critical risk. Escalate only from recorded failure or explicit quality-first authority.

The schema-v3 recommendation helper, schema-4 role templates, and route-matrix tests define the optional evaluation defaults. Production routing remains autonomous and this note does not constrain root judgment.

The key evaluation is empirical: compare total raw tokens, cost, latency, task quality, and validation on the next representative implementation task. The target is at least 50% fewer raw tokens than v0.2 across the complete task, with equivalent quality and validation. Count every adaptive workstream. Fixed system, tool, skill, environment, and repository instructions in new children are unavoidable and must be separated from router-generated context when interpreting results.
